# Configure Password Reset Email in Supabase

## Important: No Edge Function Needed!

Password reset is a **built-in feature** of Supabase Auth. You don't need to create an Edge Function for it. The `resetPasswordForEmail()` method automatically uses Supabase's authentication system.

## What You Need to Configure

### 1. Email Templates (In Supabase Dashboard)

Go to: **Authentication** → **Email Templates** → **Change Password**

#### Check the Template:

The default template should look something like this:

```html
<h2>Reset Password</h2>

<p>Follow this link to reset the password for your user:</p>
<p><a href="{{ .SiteURL }}/auth/callback?token_hash={{ .TokenHash }}&type=recovery&next={{ .RedirectTo }}">Reset Password</a></p>
```

#### Important Notes:

- The `{{ .SiteURL }}` variable is set in your Site URL configuration
- The `{{ .TokenHash }}` is automatically generated by Supabase
- The `{{ .RedirectTo }}` is what you pass in the `redirectTo` parameter

### 2. Site URL Configuration

Go to: **Authentication** → **URL Configuration**

#### Set Your Site URL:

For local development: `http://localhost:19006`

For your mobile app in production, you might leave this as your project URL, but use redirect URLs for the actual redirects.

### 3. Redirect URLs

Go to: **Authentication** → **URL Configuration** → **Redirect URLs**

#### Add These URLs:

```
http://localhost:19006/auth/callback
tradeflow://auth/callback
```

Add one per line, then click "Add URL" or "Save".

## Testing the Configuration

### 1. Check if Emails Are Enabled

Go to: **Settings** → **Authentication** → **SMTP Settings**

- If using Supabase's default email service, it should just work
- If you've configured custom SMTP, make sure it's set up correctly

### 2. Check Email Rate Limits

Go to: **Settings** → **Authentication**

Look for rate limiting settings. Default is usually fine, but if you're testing a lot, you might hit limits.

### 3. Test the Password Reset

1. In your app, enter an email that exists in Supabase Auth
2. Click "Forgot password?"
3. **Check the logs** in Supabase Dashboard:
   - Go to **Authentication** → **Logs** (or **Logs** in sidebar)
   - You should see an entry for password reset email sent

4. Check your email (including spam folder!)

## Common Issues

### Issue: No email received

**Check:**
1. Email exists in Supabase Auth users
2. Email is verified (or email confirmation is disabled)
3. SMTP is configured correctly
4. Check rate limits
5. Check spam folder

### Issue: Email received but link doesn't work

**Check:**
1. Redirect URLs are configured correctly
2. The dev server is running on the expected port
3. The platform detection in the code is working

### Issue: "Invalid credentials" or similar error

**Check:**
1. Token hasn't expired (default 1 hour)
2. Token hasn't been used already (one-time use)
3. User still exists in the system

## How It Works (No Edge Function Needed)

```
Your App                    Supabase Auth                     User's Email
   |                              |                                 |
   | resetPasswordForEmail()      |                                 |
   |----------------------------->|                                 |
   |                              |                                 |
   |                              | Generate token                  |
   |                              | Create reset link               |
   |                              |                                 |
   |                              | Send email via SMTP             |
   |                              |-------------------------------->|
   |                              |                                 |
   |<------Success response-------|                                 |
   |                              |                                 |
   |                              |                          User clicks link
   |                              |<-----------Request with token---|
   |                              |                                 |
   |                              | Validate token                  |
   |                              | Create session                  |
   |                              |                                 |
   |<--Redirect to your app-------|                                 |
   |   with session established   |                                 |
```

## Verify Your Configuration

### Quick Checklist:

- [ ] Site URL is set (e.g., `http://localhost:19006`)
- [ ] Redirect URLs include both:
  - [ ] `http://localhost:19006/auth/callback`
  - [ ] `tradeflow://auth/callback`
- [ ] Email template exists for "Change Password"
- [ ] SMTP is configured (or using default Supabase email)
- [ ] User exists in Authentication → Users
- [ ] Rate limits not exceeded

## Still Having Issues?

### Check Supabase Logs

1. Go to **Logs** in Supabase Dashboard
2. Filter by "Auth" or search for your email
3. Look for errors or email send attempts

### Check Browser Console

When testing in browser dev mode:
1. Open browser DevTools (F12)
2. Look at Console tab for errors
3. Look at Network tab to see if the API call succeeds

### Check the Response

The `resetPasswordForEmail()` call should return success even if the email doesn't exist (for security). Check the actual response:

```javascript
const { data, error } = await supabase.auth.resetPasswordForEmail(email, { redirectTo });
console.log('Reset response:', { data, error });
```

If there's an error, it will tell you what's wrong.
